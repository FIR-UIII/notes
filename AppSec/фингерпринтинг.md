На основе предоставленных источников, ниже приведено описание технологий Canvas, WebGL и AudioContext в контексте их работы и использования для идентификации пользователей (фингерпринтинга)
В совокупности эти три технологии позволяют веб-сайтам формировать высокоточный профиль пользователя для целей антифрода, защиты от ботов или рекламного отслеживания.

### **Canvas**
**Canvas** — это технология, предназначенная для **2D-рендеринга** графики и текста непосредственно в браузере. 
*   **Как это работает:** Скрипты создают невидимый элемент холста, на котором отрисовываются сложные изображения, фигуры и текстовые строки (например, панграммы, содержащие все буквы алфавита). Затем браузер извлекает пиксельные данные этого изображения и преобразует их в уникальный хэш. В итоге Canvas-хэш содержит в себе информацию о ОС + движок браузера + версия браузера + GPU + конфигурация шрифтов устройства.
*   **Почему это уникально:** Хэш Canvas зависит от сочетания операционной системы, движка браузера, видеокарты и установленных в системе шрифтов. Даже небольшие различия в алгоритмах сглаживания (например, Windows ClearType против macOS CoreText) или версии системных библиотек приводят к тому, что одно и то же изображение на разных устройствах будет отличаться на уровне пикселей.
```
(async () => {
  const canvas = document.createElement('canvas');
  canvas.width = 200;
  canvas.height = 50;
  canvas.style.display = 'none';
  const ctx = canvas.getContext('2d');
  if (!ctx) {
    console.error('Canvas context not available');
    return;
  }
  ctx.textBaseline = 'top';
  ctx.font = '20px Arial';
  ctx.fillStyle = '#f60';
  ctx.fillText('Fingerprint', 10, 10);
  ctx.fillStyle = 'rgb(102, 204, 0)';
  ctx.fillRect(125, 1, 62, 20);
  const imageData = ctx.getImageData(0, 0, canvas.width, canvas.height);
  const hashBuf = await crypto.subtle.digest(
    'SHA-256',
    imageData.data.buffer
  );
  const hash = Array.from(new Uint8Array(hashBuf))
    .map(b => b.toString(16).padStart(2, '0'))
    .join('');
  console.log('Canvas hash:', hash);
})();
```

### **WebGL**
**WebGL** — это программный интерфейс (API) для работы с **3D-графикой**, который, в отличие от Canvas, предоставляет более прямой доступ к «железу» — **графическому процессору (GPU)** и его драйверам.
*   **Что он раскрывает:** Технология позволяет сайтам получить информацию о вендоре и модели видеокарты, версии драйверов, а также список поддерживаемых расширений API. Мобильные GPU ещё более отличимы: Qualcomm Adreno, ARM Mali, Apple Bionic — каждая со своим набором возможностей.
*   **Высокая уникальность:** WebGL дает больше информации, чем Canvas, так как параметры рендеринга зависят от микроархитектуры GPU и точности вычислений шейдеров. Сайты могут идентифицировать устройство даже по специфическим артефактам отрисовки 3D-сцен или по времени выполнения «тяжелых» графических задач.
```
(() => {
  const canvas = document.createElement('canvas');
  canvas.width = 256;
  canvas.height = 256;

  const gl =
    canvas.getContext('webgl2') ||
    canvas.getContext('webgl') ||
    canvas.getContext('experimental-webgl');

  if (!gl) {
    console.log('WebGL не поддерживается');
    return;
  }

  // Базовая информация
  const vendor = gl.getParameter(gl.VENDOR);
  const renderer = gl.getParameter(gl.RENDERER);
  console.log('Vendor:', vendor);
  console.log('Renderer:', renderer);

  // Реальная информация GPU
  const debugInfo = gl.getExtension('WEBGL_debug_renderer_info');

  if (debugInfo) {
    const unmaskedVendor = gl.getParameter(debugInfo.UNMASKED_VENDOR_WEBGL);
    const unmaskedRenderer = gl.getParameter(debugInfo.UNMASKED_RENDERER_WEBGL);

    console.log('Unmasked Vendor:', unmaskedVendor);
    console.log('Unmasked Renderer:', unmaskedRenderer);
  } else {
    console.log('WEBGL_debug_renderer_info недоступно');
  }
  // Capabilities
  const maxTexSize = gl.getParameter(gl.MAX_TEXTURE_SIZE);
  const maxViewport = gl.getParameter(gl.MAX_VIEWPORT_DIMS);
  const glslVersion = gl.getParameter(gl.SHADING_LANGUAGE_VERSION);
  console.log('Max texture size:', maxTexSize);
  console.log('Max viewport dims:', maxViewport);
  console.log('GLSL version:', glslVersion);
})();
```

**AudioContext**
**AudioContext** (часть WebAudio API) — это инструмент для генерации, обработки и анализа **аудиосигналов** в браузере.
*   **Механизм сбора данных:** Система создает «оффлайн» аудиоконтекст, генерирует звуковой сигнал (например, треугольную волну определенной частоты), пропускает его через компрессоры или фильтры и анализирует итоговый буфер данных. На выходе получается числовое значение или хэш, который служит «звуковым отпечатком».
*   **Особенности:** В отличие от графических параметров, аудио-отпечаток обычно одинаков для разных браузеров на одном компьютере. Его уникальность обусловлена особенностями **аудиодрайверов операционной системы** и точностью вычислений процессора (float32). Это делает его стабильным идентификатором именно устройства, а не конкретной программы-браузера.
```
(async () => {
  const AC = window.OfflineAudioContext || window.webkitOfflineAudioContext;
  if (!AC) {
    console.log('AudioContext не поддерживается');
    return;
  }

  // Оффлайн аудиоконтекст: 1 канал, 5000 сэмплов, 44100 Hz
  const ctx = new AC(1, 5000, 44100);

  // Генератор синусоиды
  const osc = ctx.createOscillator();
  osc.type = 'triangle';      // треугольная волна
  osc.frequency.value = 1000; // 1 кГц

  // Компрессор для усиления различий
  const comp = ctx.createDynamicsCompressor();
  comp.threshold.value = -50;
  comp.knee.value = 40;
  comp.ratio.value = 12;
  comp.attack.value = 0;
  comp.release.value = 0.25;

  // Соединяем цепь
  osc.connect(comp);
  comp.connect(ctx.destination);

  // Генерируем сигнал
  osc.start(0);
  const rendered = await ctx.startRendering();

  // Буфер сэмплов
  const samples = rendered.getChannelData(0);

  // Простой отпечаток: сумма абсолютных значений
  let sum = 0;
  for (let i = 0; i < samples.length; i++) {
    sum += Math.abs(samples[i]);
  }
  const audioFingerprint = Math.round(sum * 1e6) / 1e6;
  console.log('Audio Fingerprint:', audioFingerprint);

  // Альтернатива: хеширование
  const hashBuf = await crypto.subtle.digest(
    'SHA-256',
    new Uint8Array(samples.buffer)
  );
  const hash = Array.from(new Uint8Array(hashBuf))
    .map(b => b.toString(16).padStart(2, '0'))
    .join('');
  console.log('Audio SHA-256:', hash);
})();
```

### Что еще есть
Font enumeration (DOM) - Перебором проверяет наличие известных шрифтов (через измерение текста). Каждая система имеет уникальный набор установленных шрифтов.
Navigator/UA hints - Считывает свойства navigator: userAgent, platform, deviceMemory, языки, кол-во потоков CPU, плагины, MimeTypes, Touch support, Battery API и др.
Client Hints (Sec-CH-UA) - Запрашивает у браузера доп. заголовки: модель устройства (Sec-CH-UA-Model), версия ОС (Sec-CH-UA-Platform-Version) и др
WebRTC leak (ICE) - Пытается получить локальный IP-адрес через WebRTC ICE (например, RTCPeerConnection и candidate строки).
Персистентность идентиф. - Использует суперкуки типа Evercookie, IndexedDB, localStorage, Service Workers, чтобы пометить браузер.

### Сценарии использования
Сценарий 1: Детект мультиаккаунтинга в финтехе. Пользователь открывает личный кабинет. Система собирает Canvas+WebGL+Audio, а также IP, TLS-отпечаток, поведение. Если через час с тем же Canvas/WebGL профилем, но с другого IP регистрируется новый аккаунт, система посчитает это попыткой обхода ограничений (например, создания второго аккаунта одним пользователем).
Сценарий 2: Выявления ботов или headless скриптов для срейпинга

### Для специалистов по антифроду
Не полагайтесь на один хеш Canvas.
Снимайте Canvas-отпечаток до и после критических действий (например, перед логином и после).
Проверяйте стабильность: если у одного пользователя хеш резко поменялся за короткий срок — сигнал риска (может указывать на эмулятор или защиту).
Смотрите в совокупности: Если Canvas стабилен, но WebGL выглядит замаскированным есть подозрения что что-то не так.
Анализируйте комплексно через ML.
Не стройте простых правил только на Canvas-хеше.
Обучайте модель на векторе признаков: (Canvas, WebGL, User-Agent, тайминги, сеть и т.д.).
Добавляйте поведенческие фичи: скорость взаимодействия, паттерны кликов, порядок событий.
Проверяйте консистентность параметров.
User-Agent говорит Windows 10, а аудио-отпечаток соответствует macOS (несостыковка).
Canvas указывает на RTX 3080, а WebGL — на Intel Iris (подмена).
Системные шрифты латинские, а заголовок Accept-Language: Chinese (вероятно, спуфинг).
Отслеживайте «эволюцию» устройства.
Норма: Canvas-хеш чуть изменяется (на доли процента) раз в 2-4 недели (обновление драйверов, ОС).
Аномалия: Canvas совсем не меняется 90+ дней (может указывать на виртуалку/эмулятор или очень старое ПО) или, наоборот, меняется слишком часто.

### Как найти
Если вы изучаете исходный код страницы или используете инструменты разработчика в браузере, ищите следующие технические признаки:
Для Canvas: создание невидимого элемента document.createElement('canvas'), вызовы методов toDataURL или getImageData для извлечения пикселей. Часто для этого используются строки-панграммы, такие как "Cwm fjordbank glyphs mute quiz".
Для WebGL: вызовы getContext('webgl') или getContext('webgl2'), а также использование расширения WEBGL_debug_renderer_info для получения данных о видеокарте.
Для AudioContext: создание OfflineAudioContext, использование методов createOscillator, createDynamicsCompressor и вызов startRendering для генерации аудио-хэша
